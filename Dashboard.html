How It Works

1. The **VigilKeeper** monitors all your running keepers in real-time.

2. If a keeper fails, the assistant immediately analyzes the **execution logs** and identifies the core problem using AI.

3. It provides a clear diagnosis and a suggested **fix**, helping you solve the problem in seconds instead of hours.

Critical Error: Aave Position Liquidation (0x27...a6d)

AI Diagnosis

The transaction failed with an "out of gas" error. This indicates that the gas limit set for the keeper was insufficient to execute the Aave `flashLoan` function under current network conditions. The recent spike in network activity increased the required gas.

Suggested Fix
  • **Increase Gas Limit:** Increase the `gasLimit` parameter for this Keeper from 500,000 to **750,000**.
  • **Use Gas Optimizations:** Enable KeeperHub's built-in intelligent gas optimization feature for this Keeper to automatically adjust to market conditions.
